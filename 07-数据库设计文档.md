### 0.3 通用约定

- **主键**：所有表主键为 `BIGINT AUTO_INCREMENT`，命名 `xxx_id`
- **时间戳**：除 `behavior_trace` / `communication_log` / `setting_change` / `user_setting` / `audit_log` 外，其余表继承 `TimestampMixin`，含 `created_at` / `updated_at`
- **时间存储**：UTC 时间，应用层用 `datetime.now(timezone.utc)`
- **软删**：仅 `long_term_memory` 有 `is_deleted` 字段，其余表物理删除
- **布尔**：MySQL 无 BOOL，统一用 `TINYINT`（0/1）
- **枚举**：用 `VARCHAR` + 注释说明取值，不用 MySQL ENUM（便于扩展）
- **JSON 字段**：MySQL 原生 JSON 类型，存储结构化数据

## 1. 用户体系

### 1.1 wx_account（微信账号绑定）

> openid/unionid 与系统用户的桥接表。微信登录后必建一条。

| 字段 | 类型 | 可空 | 默认 | 索引 | 说明 |
|------|------|:----:|------|:----:|------|
| account_id | BIGINT AI | 否 | - | PK | 账号 ID |
| openid | VARCHAR(64) | 否 | - | UK | 小程序 openid（唯一） |
| unionid | VARCHAR(64) | 是 | NULL | IDX | 开放平台 unionid（跨小程序识别） |
| session_key | VARCHAR(128) | 是 | NULL | - | 会话密钥（加密存储，用于解密手机号等） |
| user_type | VARCHAR(8) | 否 | - | - | 账号类型：`user` / `child` |
| ref_user_id | BIGINT | 是 | NULL | - | 关联的业务 ID（user.user_id 或 child_user.child_id） |
| last_login_at | DATETIME | 是 | NULL | - | 最近登录时间 |
| created_at | DATETIME | 否 | now() | - | 创建时间 |
| updated_at | DATETIME | 否 | now() | - | 更新时间 |

**关系**：
- 1:1 → `user`（user_type=user 时，ref_user_id 指向 user.user_id）
- 1:1 → `child_user`（user_type=child 时，ref_user_id 指向 child_user.child_id）
- 1:N → `subscribe_grant`（一个账号可授权多个订阅模板）

### 2.2 long_term_memory（长期记忆 = RAG 知识库）

> 跨会话记忆条目，既是用户记忆也是 RAG 知识库。向量存 Chroma，MySQL 存 vector_id 外键。

| 字段 | 类型 | 可空 | 默认 | 索引 | 说明 |
|------|------|:----:|------|:----:|------|
| memory_id | BIGINT AI | 否 | - | PK | 记忆 ID |
| user_id | BIGINT | 是 | NULL | IDX→FK | 老人 ID（通用知识可空）→ user.user_id |
| memory_type | VARCHAR(32) | 否 | - | - | 类型：用药/健康/偏好/医嘱/通用 |
| source | VARCHAR(16) | 否 | 'dialog' | - | 来源：`dialog`（对话抽取）/ `admin`（手动）/ `system`（系统） |
| content | TEXT | 否 | - | - | 记忆内容 |
| summary | VARCHAR(255) | 是 | NULL | - | 摘要 |
| vector_id | VARCHAR(64) | 是 | NULL | IDX | Chroma 向量库对应 ID |
| importance | TINYINT | 否 | 3 | - | 重要度：1-5（5最重要） |
| source_msg_id | BIGINT | 是 | NULL | FK | 来源消息 ID → message.msg_id（dialog 时） |
| is_deleted | TINYINT | 否 | 0 | - | 软删：0正常 / 1已删 |
| created_at | DATETIME | 否 | now() | - | 创建时间 |
| updated_at | DATETIME | 否 | now() | - | 更新时间 |

**业务规则**：
- `user_id` 为空表示通用知识（所有老人共享）
- 删除时软删（is_deleted=1），不同步删 Chroma（异步清理）
- 写入时同步写 Chroma，拿到 vector_id 回填

## 10. 初始化与维护

### 10.1 建库
```sql
CREATE DATABASE IF NOT EXISTS ftraining_db
  CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 10.2 建表（dev 模式）
```bash
cd backend
PYTHONUTF8=1 venv/Scripts/python -m app.init_db
```
执行 `Base.metadata.create_all()`，已存在的表跳过；同时灌入 dev 种子数据：
- admin_user：admin/admin（SUPER）
- api_config：default（默认大模型）
- prompt_template：default（默认系统提示词）

```bash
cd backend
PYTHONUTF8=1 venv/Scripts/alembic revision --autogenerate -m "描述"
PYTHONUTF8=1 venv/Scripts/alembic upgrade head
```

### 10.4 审计日志清理
定时任务每天执行：
```sql
DELETE FROM audit_log WHERE created_at < DATE_SUB(NOW(), INTERVAL 180 DAY);
```

mysqldump -u root -p ftraining_db > ftraining_db_$(date +%Y%m%d).sql
