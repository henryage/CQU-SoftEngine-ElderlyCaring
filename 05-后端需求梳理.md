## 5. 关键技术要点

### 5.1 微信小程序登录链路
```
小程序 wx.login() → code
  → POST /auth/wx-login {code}
  → 服务端 code2session(code) → openid + session_key
  → 查/建 wx_account → 关联 user 或 child_user
  → 签发 JWT（account_id, user_type, ref_id）
  → 返回 {token, user_info}
```

### 5.2 流式问答（WebSocket）+ 大模型预警触发
- 小程序 `wx.connectSocket` 连 `/qa/stream/{task_id}`
- 服务端 FastAPI WebSocket 代理大模型 chunk
- **大模型回答末尾追加结构化预警信号**（JSON schema 约束输出）：
  ```json
  {"type":"alert_signal","alert_type":"fall","alert_level":"urgent","detail":"老人提到跌倒"}
  ```
- 后端解析 → 写 `alert_event(trigger_source=llm)` → 推送子女端 WebSocket
- 同时推送小猫动作状态 + 合规拦截事件
- 断线基于 task_id 续传

### 5.3 RAG 记忆（Chroma）
- 写入：消息落库 → 异步抽取记忆 → Chroma 向量化入库 → MySQL `long_term_memory` 落库（存 vector_id）
- 召回：当前问题向量化 → Chroma top-K → 拼接进 Prompt
- 记忆库即知识库，不再单独维护知识包

### 5.4 预警触发机制（三类）
| 类型 | 触发源 | 场景 |
|------|--------|------|
| 对话类 | 大模型（llm） | 跌倒、情绪低落、用药异常（问答中识别） |
| 定时类 | 规则（rule） | 无活动>12h、用药提醒未响应、定时问候、**网络断开** |
| 主动类 | 老人端（manual） | 紧急呼叫一键按钮 |
| 恢复类 | 规则（rule） | **网络恢复**（老人重新心跳时触发） |

### 5.4.1 老人端网络断开自动预警（★ 新增）
**场景**：老人端小程序因网络故障、手机关机、小程序被杀后台等原因断开连接，子女端需要及时知晓。

**机制**：
1. 老人端小程序每 **30 秒** 调 `POST /auth/heartbeat` 上报心跳
2. 后端更新 `user.last_heartbeat_at` 与 `online_status=online`
3. 后台定时任务（每 60 秒扫描）检查所有 `online_status=online` 但 `last_heartbeat_at` 超过 **180 秒**（3 分钟）未更新的老人
4. 命中则：
   - 改 `online_status=offline`
   - 写 `alert_event(alert_type=网络断开, alert_level=警告, trigger_source=rule)`
   - 推送子女端 `alert.warn` 订阅消息
   - 去重：同一老人有待处理的"网络断开"预警时不重复发
5. 老人重新心跳时：
   - 检测到 `was_offline=True`
   - 写 `alert_event(alert_type=网络恢复, alert_level=提醒, handling_status=已确认)`
   - 推送子女端 `alert.handled` 订阅消息

**字段新增**（user 表）：
- `last_heartbeat_at DATETIME` — 最后心跳时间，索引
- `online_status VARCHAR(16)` — online/offline，默认 offline

**接口新增**：
- `POST /auth/heartbeat` — 老人端定时调用，仅 USER 角色可调

### 5.5 鉴权与权限
- JWT（Access 30min + Refresh 7d）
- 三角色 RBAC：USER / CHILD / ADMIN
- 子女端访问老人数据需校验 `user_child_relation` 绑定关系
- 高危操作二次密码验证

### 5.6 适老化配置下发
子女端 PUT → 写 `setting_change`（待同步）+ 更新 `user_setting` → 老人端下次启动 / WebSocket 推送 → 应用 → 回执 → `sync_status=已同步`

### 5.7 主动推送（订阅消息）
- 定时类推送走微信订阅消息（用药、问候、预警、留言）
- 用户需在小程序内授权 → 上报 `subscribe_grant`
- 一次性模板用完需重新授权；配额不足时降级短信通知子女

### 5.8 紧急预警链路
```
老人端一键 / 大模型识别 → 写 alert_event(紧急)
  → 推送子女端订阅消息
  → 30s 未确认 → 短信
  → 60s 未确认 → 电话
  → 全程写 communication_log + audit_log + notify_channels
```

### 5.9 数据导出
- 服务端生成导出包（CSV + JSON 双格式）→ 上传对象存储 → 临时 URL → 邮件 / 微信文件助手

### 5.10 定时任务
APScheduler 或 Celery Beat：用药提醒、问候、无活动监测、情绪低落定期回顾、**老人离线检测（每60秒扫描）**

### 5.11 老人端心跳与离线检测（★ 新增）
- 老人端小程序前台运行时每 30 秒调 `POST /auth/heartbeat`
- 后端 `app/tasks/monitor.py` 每 60 秒扫描，超 180 秒未心跳判为离线
- 离线触发预警 → 推送子女端；恢复心跳触发"网络恢复"通知
- 详见 5.4.1 节
