# 管理端接口文档

> 模糊视觉辅助问答系统 — 管理端 Web 控制台  
> 服务地址：`http://10.242.5.159:8090` | Swagger：`http://10.242.5.159:8090/docs`

---

## 1. 基础约定

### 1.1 鉴权

管理员登录获取 token，此后所有 `/admin/*` 接口携带 `Authorization: Bearer <admin_token>`。

dev 管理员：`admin / admin`，通过 `GET /auth/dev/create-admin` 创建。

### 1.2 统一响应

```json
{"code": 0, "msg": "ok", "data": {...}}
```

---

## 2. 接口清单（10 个）

| # | 方法 | 路径 | 说明 |
|:--:|------|------|------|
| 1 | POST | `/auth/admin/login` | 管理员登录 |
| 2 | GET | `/admin/users` | 老人用户列表 |
| 3 | GET | `/admin/children` | 子女账号列表 |
| 4 | GET | `/admin/relations` | 绑定关系查询（双向） |
| 5 | GET | `/admin/audit` | 审计日志 |
| 6 | GET | `/admin/audit/export?format=csv` | 审计导出 |
| 7 | GET | `/admin/subscribe/templates` | 订阅模板列表 |
| 8 | POST | `/admin/subscribe/templates` | 新增订阅模板 |
| 9 | GET | `/alert?user_id=6` | 预警列表（复用） |
| 10 | PATCH | `/alert/{id}/handle` | 处置预警（复用） |

> 管理端 token 可调所有接口（含子女端/老人端，权限最高）。

## 3. 接口详情

### 3.1 管理员登录 `POST /auth/admin/login`

```json
// 请求
{"username": "admin", "password": "admin"}
// 响应
{"code":0,"data":{"token":"eyJ...","user_type":"admin","ref_id":1,"nickname":"admin"}}
```

### 3.2 老人用户列表 `GET /admin/users?page=1&keyword=老人`

```json
{"code":0,"data":{"total":23,"items":[
  {"user_id":6,"nickname":"妈妈","online_status":"online","last_heartbeat_at":"2026-06-30T08:30:00Z","status":1,"created_at":"2026-06-27T00:00:00Z"}
]}}
```

| Query | 说明 |
|-------|------|
| page/page_size | 分页，默认 50/页 |
| keyword | 模糊搜索昵称 |

### 3.3 子女账号列表 `GET /admin/children`

```json
{"code":0,"data":{"total":5,"items":[
  {"child_id":9,"name":"子女9","phone":"dev_child_001","created_at":"2026-06-30T00:00:00Z"}
]}}
```

### 3.4 绑定关系 `GET /admin/relations?user_id=6` 或 `?child_id=9`

双向查询，必须传 `user_id` 或 `child_id` 之一。

```
GET /admin/relations?user_id=6    → 查妈妈绑了哪些子女
GET /admin/relations?child_id=9   → 查子女绑了哪些老人
```

```json
{"code":0,"data":{"count":2,"items":[
  {"relation_id":1,
   "elderly":{"user_id":6,"nickname":"妈妈","online_status":"online"},
   "child":{"child_id":9,"name":"小明","phone":"138..."},
   "relation":"母亲","is_primary":true,
   "created_at":"2026-06-30T00:00:00Z"}
]}}
```

### 3.5 审计日志

| Query | 说明 |
|-------|------|
| operation | 操作类型：login/ask/bind 等 |
| operator_type | user/child/admin/system |

### 3.6 审计导出 `GET /admin/audit/export?format=csv`

返回文件下载（CSV 或 JSON），最多 10000 条。

### 3.7 订阅模板 `GET /admin/subscribe/templates`

```json
{"code":0,"data":[
  {"template_id":"tpl_medication","code":"medication.remind","name":"用药提醒","status":1}
]}
```

### 3.8 新增模板 `POST /admin/subscribe/templates`

```
?template_id=tpl_new&code=alert.urgent&name=紧急预警&scene=老人紧急呼叫通知
```

### 3.9 预警列表 `GET /alert?user_id=6&status=待处理`

复用子女端预警接口，管理端可直接查看任意老人的预警。

### 3.10 处置预警 `PATCH /alert/{alert_id}/handle?handling_status=已确认&handle_remark=管理员确认`

相同接口，管理端可代为处置。

## 4. 功能流程

### 4.1 用户管理

```
登录 → GET /admin/users?keyword=妈妈
  → 查看用户列表
  → 点击某用户 → 可结合子女端接口查看该用户的看板/问答/配置
```

### 4.2 审计追踪

```
GET /admin/audit?operation=login&page=1
  → 分页查看操作轨迹
  → GET /admin/audit/export?format=csv → 下载完整日志
```

### 4.3 预警处置

```
看板发现有预警
  → GET /alert?user_id=6&status=待处理
  → PATCH /alert/5/handle?handling_status=已确认
```
