# 06 - 小程序类目与权限梳理

> 配套《05-后端需求梳理 v3》· 老人端 / 子女端两个独立微信小程序
> 编制：Senior Developer · 2026-06-27

---

## 1. 小程序主体与类目选择

### 1.1 主体选择

| 项 | 推荐 | 说明 |
|----|------|------|
| 主体类型 | **企业主体**（或个体工商户） | 个人主体不支持 `getPhoneNumber`、不支持长期订阅消息、类目受限 |
| 主体资质 | 营业执照 | 老人端、子女端可注册在同一主体下，或分别注册两个独立小程序 |

### 1.2 类目选择（老人端）

> 核心矛盾：系统涉及"健康画像/用药/预警"，但"医疗"类目需医疗机构资质，审核严。建议**规避医疗类目**，定位为"健康辅助工具"。

| 推荐类目 | 一级 | 二级 | 是否需资质 | 用途 |
|---------|------|------|----------|------|
| **主类目①** | 工具 | 效率 | 否 | 通用问答、截图识别、语音输入 |
| **主类目②** | 生活服务 | 便民服务 | 否 | 用药提醒、问候、紧急呼叫 |
| 备选 | 教育 | 在线教育 | 否（部分需ICP） | 健康知识科普口径 |

**规避类目**：
- ❌ 医疗 → 互联网医院/医药（需《医疗机构执业许可证》等）
- ❌ 医疗 → 就医服务（需医院主体或合作授权）

**定位话术**：本系统为"健康辅助工具"，不提供医疗诊断，所有涉医内容附加免责声明，由合规拦截看门狗处理。

### 1.3 类目选择（子女端）

| 推荐类目 | 一级 | 二级 | 是否需资质 | 用途 |
|---------|------|------|----------|------|
| **主类目①** | 工具 | 效率 | 否 | 健康看板、轨迹、画像查看 |
| **主类目②** | 生活服务 | 便民服务 | 否 | 配置下发、通信、预警处置 |

> 子女端不直接对老人提供"医疗服务"，仅展示数据 + 通信 + 配置，工具类目即可覆盖。

### 1.4 实时音视频类目

> 本项目已去掉视频通话，**不需要**实时音视频类目（`<live-pusher>`/`<live-player>` 权限）。如果未来恢复视频通话，需补充"社交-直播"或"教育-在线教育"类目并申请 live-pusher 组件权限。

## 2. 权限矩阵

### 2.1 用户授权类（scope，需用户点击按钮授权）

| 权限 | scope 值 | 老人端 | 子女端 | 用途 | 触发时机 |
|------|---------|:------:|:------:|------|---------|
| 用户信息 | `getUserProfile` | ✓ | ✓ | 昵称头像（首次登录） | 首次进入引导页 |
| 手机号 | `getPhoneNumber` | ✗ | ✓ | 子女绑定手机号 | 子女注册绑定流程 |
| 录音 | `scope.record` | ✓ | ✗ | 语音问答输入 | 首次点击语音按钮 |
| 相机 | `scope.camera` | ✓ | ✗ | 拍照问答 | 首次点击拍照按钮 |
| 位置 | `scope.userLocation` | 可选 | ✗ | 紧急呼叫附带位置（可选） | 紧急呼叫时 |

### 2.2 订阅消息类（subscribeMessage，需用户授权）

| 模板 | 老人端授权 | 子女端授权 | 说明 |
|------|:---------:|:---------:|------|
| `medication.remind` | ✓ | ✗ | 老人授权接收用药提醒 |
| `alert.remind` | ✗ | ✓ | 子女授权接收提醒级预警 |
| `alert.warn` | ✗ | ✓ | 子女授权接收警告级预警 |
| `alert.urgent` | ✗ | ✓ | 子女授权接收紧急级预警 |
| `comm.text` | ✓ | ✗ | 老人授权接收子女文字留言 |
| `comm.voice` | ✓ | ✗ | 老人授权接收子女语音 |
| `greeting.scheduled` | ✓ | ✗ | 老人授权接收定时问候 |
| `setting.updated` | ✓ | ✗ | 老人授权接收配置更新通知 |
| `alert.handled` | ✓ | ✗ | 老人授权接收预警处置回执 |

> 授权策略：老人端在首次进入"提醒设置"或"通信"页时一次性引导授权所有相关模板；子女端在绑定老人后引导授权三个 alert 模板。一次性模板用完需重新授权，建议产品流程中常态化引导。

|-----|------|

## 3. 服务器域名白名单

> 在微信公众平台 → 开发管理 → 开发设置中配置。每个小程序独立配置。

### 3.1 老人端

| 类型 | 域名 | 用途 |
|------|------|------|
| request 合法域名 | `https://api.yourdomain.com` | RESTful 接口 |
| socket 合法域名 | `wss://api.yourdomain.com` | WebSocket 流式问答 + 预警 |
| uploadFile 合法域名 | `https://api.yourdomain.com` | 媒体上传 |
| downloadFile 合法域名 | `https://cdn.yourdomain.com` | 对象存储下载（图/语音/导出） |

### 3.2 子女端

| 类型 | 域名 | 用途 |
|------|------|------|
| request 合法域名 | `https://api.yourdomain.com` | RESTful 接口 |
| socket 合法域名 | `wss://api.yourdomain.com` | 实时预警推送 |
| uploadFile 合法域名 | `https://api.yourdomain.com` | 语音消息上传 |
| downloadFile 合法域名 | `https://cdn.yourdomain.com` | 导出文件下载 |

> 生产环境必须 HTTPS/WSS。开发环境可在微信开发者工具勾选"不校验合法域名"。

## 4. 订阅消息模板与授权引导流程

### 4.1 模板申请清单（共 9 个）

> 在微信公众平台 → 订阅消息 → 公共模板库选模板，或自行申请新模板。每个模板需选好"关键词"组合。

| # | 模板编码 | 推送对象 | 建议关键词 |
|---|---------|---------|-----------|
| 1 | `medication.remind` | 老人 | 药品名称、用药剂量、用药时间、备注 |
| 2 | `alert.remind` | 子女 | 老人姓名、提醒类型、提醒时间、详情 |
| 3 | `alert.warn` | 子女 | 老人姓名、预警类型、预警时间、详情、处置链接 |
| 4 | `alert.urgent` | 子女 | 老人姓名、紧急类型、触发时间、详情、立即查看 |
| 5 | `comm.text` | 老人 | 子女姓名、留言内容、留言时间 |
| 6 | `comm.voice` | 老人 | 子女姓名、语音时长、留言时间 |
| 7 | `greeting.scheduled` | 老人 | 子女姓名、问候内容、送达时间 |
| 8 | `setting.updated` | 老人 | 子女姓名、变更项目、变更时间 |
| 9 | `alert.handled` | 老人 | 子女姓名、预警类型、处置结果、处置时间 |

### 4.2 授权引导时机（产品流程设计）

**老人端首次进入引导流程**：
1. 微信登录 → 引导授权 `getUserProfile`
2. 进入首页 → 弹窗引导一次性订阅：`medication.remind` + `comm.text` + `comm.voice` + `greeting.scheduled` + `setting.updated` + `alert.handled`
3. 进入"语音问答" → 引导授权 `scope.record`
4. 进入"拍照问答" → 引导授权 `scope.camera`
5. 紧急呼叫时 → 引导授权 `alert.urgent`（如未授权）

**子女端首次绑定老人流程**：
1. 微信登录 → `getUserProfile`
2. 绑定手机号 → `getPhoneNumber`
3. 绑定老人成功后 → 弹窗引导一次性订阅：`alert.remind` + `alert.warn` + `alert.urgent`

### 4.3 配额管理

- 一次性订阅模板：每次授权 = 1 次推送配额，用完需重新授权
- 长期订阅模板：仅医疗、政务等特定类目可申请，本项目工具类目**无法申请长期订阅**
- 后端 `subscribe_grant.expire_count` 记录剩余配额，配额不足时：
  - 老人端：下次打开小程序弹屏重新引导授权
  - 子女端：降级短信通知

## 6. 风险与建议

### 6.1 风险

| 风险 | 影响 | 缓解 |
|------|------|------|
| 工具类目下"健康/用药"内容被审核驳回 | 小程序无法上线 | 所有涉医内容强制附加"非医疗诊断"免责声明；合规拦截看门狗；类目描述避免"医疗/诊断/治疗"字样 |
| 一次性订阅消息配额耗尽 | 老人收不到用药/预警推送 | 后端监控 `subscribe_grant.expire_count`；配额<3 时小程序内引导重新授权；紧急情况降级短信 |
| `getPhoneNumber` 需企业主体 | 个人主体无法用 | 注册企业主体小程序 |
| 大模型预警误报 | 子女端收到大量误报 | 后端去重/合并同类预警；设置冷却期（同类型 1h 内不重复） |
| 截图上传服务端的隐私折衷 | 老人隐私敏感数据上云 | HTTPS 传输 + 服务端敏感字段加密 + 可选私有化部署 + 审计日志 |

### 6.2 建议

1. **先注册企业主体**，再开发小程序，避免后期主体变更导致 appid 变化
2. **类目定"工具-效率"**，描述写"智能问答与生活辅助工具"，规避医疗审核
3. **订阅消息模板提前 1-2 周申请**，部分模板审核需 3-7 天
4. **服务器域名提前备案**（国内服务器需 ICP 备案）
5. **WebSocket 长连接**做好心跳 + 断线重连，小程序后台运行 30s 后 WS 可能被系统挂起
6. **紧急呼叫的 `wx.makePhoneCall`** 直接拨打子女电话，作为订阅消息推送失败的兜底通道
7. **私有化部署版本**如需支持，建议设计为可独立部署的 Docker Compose 包（FastAPI + MySQL + Chroma + MinIO）
