# 模糊视觉辅助问答系统 - 老人端微信小程序 PRD

## 1. 问题陈述

### 1.1 当前情况
当前老年用户在视力不佳的情况下，难以通过文字输入与AI进行交互，也无法方便地通过拍照识别药品、询问日常问题。现有AI应用操作复杂、字体小、交互繁琐，不适合老年人使用。

### 1.2 用户痛点
- 老人视力差，文字输入困难，打字不便
- 无法方便地通过拍照+语音方式询问药品、识别物体
- 现有AI应用界面字体小、按钮小，老人看不清、点不准
- AI回复只有文字，老人阅读困难，需要语音播报
- 用药需要提醒，容易忘记吃药
- 紧急情况下难以快速联系家人

### 1.3 业务影响
- 帮助视力不佳的老人独立获取信息和帮助
- 通过语音+视觉多模态交互降低使用门槛
- 用药提醒功能帮助老人按时服药
- 紧急呼叫功能保障老人安全
- 橘猫形象的AI助手增加亲切感，提升老人使用意愿

## 2. 方案概述

### 2.1 产品定位
打造一个面向老年用户的**模糊视觉辅助问答系统**（老人端），采用全屏相机沉浸式界面，核心交互为"按住说话，松开即问"。AI助手以橘猫形象呈现，支持语音+拍照多模态输入，AI回复自动语音播报，同时具备用药提醒和紧急呼叫功能。整体遵循适老化设计规范。

### 2.2 核心功能
1. **实时相机背景**：全屏相机预览始终作为界面背景，营造沉浸式体验，方便随时拍照
2. **语音+视觉自动输入**：按住麦克风开始录音，录音过程中自动抽取首帧、中帧、尾帧共3张图片；松开麦克风自动停止录音，自动上传语音识别文字+帧图，发送给AI问答
3. **单独拍照功能**：点击相机按钮拍一张照片，照片作为"待发送"气泡显示在对话区右侧，用户按麦克风说话后随语音一起发送
4. **橘猫AI助手**：AI以橘猫形象展示，根据状态切换图片（listen戴耳机/think思考/speak说话），回复自动语音播报
5. **对话气泡展示**：用户消息靠右、AI回复靠左（带橘猫头像），大字体、高对比度，支持文字、图片、多图混排
6. **适老化设计**：字体≥32rpx、按钮88px、大间距、高对比度、语音播报
7. **用药提醒自动推送**：每5分钟轮询后端，有新提醒时橘猫主动发送提醒消息气泡并语音播报
8. **紧急呼叫**：底部红色紧急按钮，一键通知家人并可拨打120
9. **状态反馈**：提供录音中声波动画、上传中提示、AI思考状态、发送状态等清晰反馈

### 2.3 交互设计要点
- **两种输入入口**：麦克风按钮（语音+自动抽帧）、相机按钮（单独拍照）
- **无文字键盘输入**：专注语音和视觉输入，降低老人使用门槛
- **无相册选图**：所有图片来自相机实时拍摄/抽帧
- **自动发送**：松开麦克风自动完成上传和发送，无需额外点击发送按钮
- **取消操作**：中间红色取消按钮可中止录音、清除待发送图片、中止请求
- **紧急呼叫**：底部第四个红色按钮，二次确认后触发
- **语音播报**：AI所有回复自动语音播报，点击气泡可重播

## 3. 需求规格

### 3.1 功能需求

#### FR-001：实时相机背景
**描述**：全屏相机预览始终作为界面背景层
**验收标准**：
- 进入页面即打开摄像头（后置），实时画面铺满全屏
- 相机画面上覆盖较深的半透明黑色渐变遮罩（`rgba(0,0,0,0.65)`），保证大字体文字清晰可读
- 首次进入请求相机权限，拒绝后显示权限引导提示
- 相机组件在页面生命周期中保持运行，不随页面切换销毁

#### FR-002：语音录音与自动抽帧
**描述**：按住麦克风按钮开始录音，录音过程中自动拍摄关键帧
**验收标准**：
- 按住麦克风按钮立即开始录音
- 录音开始瞬间（0s）自动拍摄首帧
- 录音时长达到一半时自动拍摄中帧
- 松开麦克风按钮时：①停止录音 ②立即拍摄尾帧 ③进入并行上传流程
- 录音最长时限60秒，超时自动停止并发送
- 录音过程中显示放大版嵌入式声波动画和"正在听..."状态文案（36rpx大字体）
- 录音时麦克风按钮高亮放大，相机按钮和紧急按钮置灰不可点击
- 录音格式：mp3/aac，采样率16kHz，单声道
- 橘猫在录音期间切换为listen状态（戴耳机图片）

#### FR-003：文件上传（语音+图片）
**描述**：录音结束后，语音文件和图片文件并行上传到后端
**验收标准**：
- 松开麦克风停止录音后，立即并行上传：
  - 语音文件上传到 `POST /api/v1/media/upload/voice`，获取`asr_text`（语音识别文字）
  - 每张图片（3张抽帧+1张待发送图片如有）依次上传到 `POST /api/v1/media/upload/image`，获取每张的`enhanced_url`
- 上传过程中状态文案显示"正在上传..."
- 上传完成后识别文字显示在状态区域，供老人确认
- 上传失败时显示错误提示，允许重试
- 橘猫在上传/思考期间切换为think状态（低头思考图片）

#### FR-004：智能问答
**描述**：上传完成后，自动调用问答接口获取AI回复
**验收标准**：
- 上传完成后，用户消息（含图片缩略图+识别文字）立即显示在右侧用户气泡中，状态为"发送中"
- 调用 `POST /api/v1/qa/ask`，参数：
  - `input_type`: "image"（有图片时）或 "text"（仅语音时）
  - `text`: asr_text语音识别文字
  - `media_url`: 第一张图片的enhanced_url（有图片时）
  - `session_id`: 当前会话ID（首次不传，后续传返回值）
- 橘猫保持think状态，显示"喵喵正在思考..."
- 收到回复后：
  - 橘猫切换为speak状态（张嘴说话图片）
  - AI回复文字显示在左侧气泡，带橘猫头像
  - 自动触发语音播报回复内容
  - 若`intercepted=true`或有`risk_tags`，气泡下方显示"⚠️ 以上内容仅供参考，请遵医嘱"
- 发送失败时用户消息旁显示红色感叹号，点击可重试

#### FR-005：单独拍照功能
**描述**：点击相机按钮单独拍摄一张照片
**验收标准**：
- 点击相机按钮直接拍照（不弹出ActionSheet，无相册选项）
- 拍照后图片以"待发送"虚线气泡形式出现在对话区右侧
- 待发送气泡视觉上与正式消息区分（虚线边框/半透明/标注"待发送"），标注"已拍照，按住麦克风说话发送"
- 若已有待发送图片，再次点击相机按钮则替换为新拍的照片（只保留一张单独拍照的图片）
- 待发送图片不立即上传，等用户按麦克风说话后与抽帧图片合并上传发送
- 点击取消按钮可清除待发送图片

#### FR-006：对话气泡列表
**描述**：以聊天对话形式展示用户消息和AI回复
**验收标准**：
- 使用scroll-view实现纵向滚动
- 用户消息靠右对齐，AI回复靠左对齐（带80x80px橘猫头像）
- 新消息到达时自动滚动到底部
- 用户气泡：白色半透明背景（`rgba(255,255,255,0.9)`）+毛玻璃，右下角小圆角，深灰文字
- AI气泡：深灰半透明背景（`rgba(30,30,30,0.85)`）+毛玻璃，左下角小圆角，纯白文字
- 气泡内字体≥32rpx，文字阴影加重（`2px 3px 6px rgba(0,0,0,0.6)`）
- 支持纯文字、纯图片、文字+图片混排、多图展示
- 多图以网格形式排列在气泡内，每张图片80x80px圆角（适老化放大）
- 发送中状态：气泡旁显示小加载圈
- 发送失败：显示红色感叹号，点击可重试
- 医疗风险提示：气泡下方显示黄色警示文字

#### FR-007：橘猫状态动画
**描述**：AI橘猫头像根据交互状态切换不同图片
**验收标准**：
- 橘猫头像位于AI气泡左侧，80x80px圆角矩形
- 三种状态图片：
  - listen（戴耳机）：用户录音期间显示
  - think（低头思考）：上传期间、AI思考期间显示
  - speak（张嘴说话）：待机状态、AI回复完成后显示
- 状态切换有平滑过渡效果
- 待机默认显示speak状态

#### FR-008：语音播报
**描述**：AI回复内容自动语音播报
**验收标准**：
- AI回复文字显示后，自动开始语音播报
- 播报期间橘猫保持speak状态
- 点击AI气泡可重播语音
- 录音开始时停止当前播报
- 语音播报音量足够大，适合老人听力

#### FR-009：取消操作
**描述**：中间红色取消按钮用于中止当前操作
**验收标准**：
- 录音中点击取消：立即停止录音，丢弃已录制音频和已抽帧，清除待发送图片，回到待机
- 待机状态下点击取消：清除当前所有待发送图片
- 上传/思考过程中点击取消：中止网络请求，回到待机
- 取消后橘猫恢复speak默认状态

#### FR-010：用药提醒自动推送
**描述**：定时轮询用药提醒接口，有新提醒时橘猫主动推送
**验收标准**：
- 小程序启动/onShow时立即调用一次 `GET /api/v1/reminder/medication`
- 之后每5分钟轮询一次该接口（后台静默调用）
- 对比上次获取的提醒列表，发现新的或到点的提醒：
  - 橘猫发送提醒消息气泡（type: 'reminder'），内容如"喵喵~8点啦，该吃硝苯地平，1片哦！"
  - 自动语音播报提醒内容
- 提醒气泡样式与普通AI回复一致，但带💊小图标
- 点击提醒气泡可重播语音

#### FR-011：紧急呼叫
**描述**：底部红色紧急呼叫按钮，一键通知家人
**验收标准**：
- 底部控制栏第四个按钮为红色紧急呼叫按钮（88px直径，`#FF3B30`背景，白色🆘图标）
- 点击后弹出二次确认弹窗："确定要通知家人吗？" [取消] [确定呼叫]（大按钮）
- 点击确定后：
  - 调用 `POST /api/v1/alert/emergency/call`
  - 弹窗显示："已通知家人，请等待。紧急情况请拨打120。"
  - 弹窗内有大按钮 [拨打120]，点击调用 `wx.makePhoneCall({ phoneNumber: '120' })`
  - 同时橘猫发送一条系统消息气泡："喵喵~已经帮您通知家人了，不要着急"
- 录音、上传、思考期间紧急按钮置灰不可点击
- 待机状态下始终可点击

#### FR-012：登录与鉴权
**描述**：启动时自动微信登录，维护token有效性
**验收标准**：
- 小程序启动时调用 `wx.login()` 获取code
- 调用 `POST /api/v1/auth/wx-login` 传入code和`user_type:"user"`，获取token、refresh_token、ref_id
- 所有接口请求Header自动携带 `Authorization: Bearer <token>`
- token过期（返回401）时，提示用户重新登录（当前接口文档未提供refresh接口）
- 登录失败显示友好提示，允许重试

#### FR-013：心跳上报
**描述**：前台运行时每30秒上报心跳
**验收标准**：
- 页面onShow时立即发送一次心跳 `POST /api/v1/auth/heartbeat`
- 启动定时器，每30秒发送一次心跳
- 页面onHide时清除定时器，暂停心跳
- onShow恢复时重新启动

### 3.2 技术需求

#### TR-001：微信小程序API使用
- 使用 `<camera>` 组件作为全屏背景，`wx.createCameraContext()` 管理相机
- 使用 `cameraContext.takePhoto()` 在录音开始、中途、结束三个时间点抽帧
- 使用 `wx.getRecorderManager()` 进行语音录制
- 录音格式配置：mp3，16kHz，单声道，码率合理控制
- 使用 `wx.uploadFile` 上传图片和语音文件到后端
- 使用 `wx.request` 调用后端HTTP接口
- 使用 `wx.makePhoneCall` 实现拨打120
- 语音播报：使用微信同声传译插件TTS或`wx.createInnerAudioContext`

#### TR-002：抽帧实现方案
- 不使用`onCameraFrame`（性能开销大、需要手动转图片）
- 采用定时拍照法：在录音开始`onStart`时拍首帧，录音时长达到`duration/2`时拍中帧，录音`onStop`时拍尾帧
- 三次拍照均使用`quality: 'normal'`以平衡画质和速度
- 三次拍照结果收集后与语音文件并行上传

#### TR-003：HTTP接口对接
- 后端基础地址：`http://127.0.0.1:8090`
- 封装统一request方法，自动携带token，处理401
- 文件上传单独封装uploadFile方法
- 所有接口遵循统一响应格式：`{code: 0, msg: "ok", data: {...}}`
- 图片URL拼接：`BASE_URL + url` 或 `BASE_URL + enhanced_url`

#### TR-004：界面实现（适老化）
- 使用微信小程序原生组件实现（WXML + WXSS + JS）
- 声波动画使用CSS3关键帧动画实现（三个圆点呼吸效果，尺寸放大）
- 底部控制栏固定定位`position: fixed`，适配全面屏安全区域（`env(safe-area-inset-bottom)`）
- 毛玻璃效果使用`backdrop-filter: blur(20px)`，降级为半透明不透明背景
- 对话列表使用`scroll-view`，通过`scroll-into-view`自动滚动到底部
- 使用rpx单位适配，**字体≥32rpx**，正文建议32-36rpx
- **按钮直径88px**，按钮间距44px，减少误触
- 文字阴影加重：`text-shadow: 0 2px 6px rgba(0,0,0,0.6)`
- 紧急呼叫按钮始终红色醒目

#### TR-005：性能要求
- 页面首次启动（含相机初始化、登录）≤ 2s
- 录音开始响应（从按下按钮到录音开始）≤ 100ms
- 拍照到获取tempPath回调 ≤ 300ms
- 松开按钮到用户消息气泡出现 ≤ 800ms（不含网络上传和AI回复耗时）
- 界面帧率 ≥ 50fps
- 相机预览不卡顿，抽帧过程不阻塞UI

### 7.3 前端网络封装

**请求基类**：
```javascript
const BASE_URL = 'http://127.0.0.1:8090';

async function request(path, options = {}) {
  const app = getApp();
  const header = {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer ' + app.globalData.token,
    ...options.header
  };
  
  return new Promise((resolve, reject) => {
    wx.request({
      url: BASE_URL + path,
      ...options,
      header,
      success: (res) => {
        if (res.statusCode === 401) {
          wx.showToast({ title: '登录已过期，请重新进入', icon: 'none' });
          reject(new Error('Unauthorized'));
          return;
        }
        if (res.data.code === 0) {
          resolve(res.data.data);
        } else {
          wx.showToast({ title: res.data.msg || '请求失败', icon: 'none' });
          reject(res.data);
        }
      },
      fail: (err) => {
        wx.showToast({ title: '网络错误', icon: 'none' });
        reject(err);
      }
    });
  });
}
```

**文件上传封装**：
```javascript
function uploadFile(filePath, type) {
  const app = getApp();
  return new Promise((resolve, reject) => {
    wx.uploadFile({
      url: BASE_URL + '/api/v1/media/upload/' + type,
      filePath: filePath,
      name: 'file',
      header: { 'Authorization': 'Bearer ' + app.globalData.token },
      success: (res) => {
        const data = JSON.parse(res.data);
        if (data.code === 0) {
          resolve(data.data);
        } else {
          reject(data);
        }
      },
      fail: reject
    });
  });
}
```

## 9. 风险与应对措施

| 风险 | 影响 | 可能性 | 应对措施 |
|-----|------|--------|---------|
| 用户拒绝相机/录音权限 | 高 | 中 | 友好的权限说明弹窗，拒绝后显示引导开启按钮；提示必需权限才能使用 |
| 拍照（takePhoto）在部分Android机型上回调慢或失败 | 中 | 中 | 设置合理超时；失败时跳过该帧不阻塞流程；多机型测试 |
| 三次连续抽帧导致相机卡顿 | 中 | 低 | 两次拍照之间留少量间隔（>100ms）；使用normal质量；异步处理上传 |
| 语音上传后ASR识别准确率低 | 高 | 中 | 识别文字显示给老人确认；允许取消重说；不阻断流程 |
| 后端服务不可用或网络错误 | 高 | 中 | 友好错误提示；支持重试；离线提示 |
| 长时间录音内存占用过高 | 中 | 中 | 设置60秒硬性上限；录音文件及时释放 |
| 全面屏/刘海屏底部按钮被遮挡 | 中 | 高 | 使用`env(safe-area-inset-bottom)`适配；多机型测试 |
| 毛玻璃backdrop-filter在低版本微信不支持 | 低 | 中 | 降级为半透明不透明背景（rgba(0,0,0,0.75)），不影响功能 |
| TTS语音播报不可用或声音小 | 中 | 中 | 预留多种TTS方案；音量默认最大；提示老人检查媒体音量 |
| 老人误触紧急呼叫按钮 | 中 | 中 | 二次确认弹窗；按钮位置合理，不易误触 |

## 10. 非功能需求

### 10.1 兼容性
- 支持微信iOS端和Android端
- 适配全面屏、刘海屏、挖孔屏（安全区域适配）
- 兼容微信基础库 2.21.0 及以上版本
- 低端机型降级动画复杂度，保证基本功能可用

### 10.2 可用性（适老化）
- **大字体**：所有文字≥32rpx，主要文字36rpx
- **大按钮**：所有可点击按钮≥88x88px，按钮间距≥44px
- **高对比度**：白色文字在深色背景上、深色文字在白色背景上，加重文字阴影
- **操作极简**：核心交互只有"按住说话"一个动作
- **反馈即时**：所有操作都有明确视觉反馈（按钮状态、放大动画、大字体文案）
- **语音播报**：AI所有回复自动语音播报，减少阅读负担
- **容错性强**：误触取消可立即中止；发送失败可重试；单帧上传失败不阻塞整体流程
- **紧急按钮醒目**：红色始终可见，二次确认防误触

### 10.3 隐私合规
- 首次使用弹框说明相机和录音用途
- 录音文件和图片仅在发送时上传到后端
- 遵守微信小程序隐私规范，在隐私协议中声明相机、录音、拨打电话权限使用
- 不在本地长期存储用户的语音和图片数据
